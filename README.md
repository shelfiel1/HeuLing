## 📌 프로젝트 개요

### 기획 배경

Heuron 의료 AI 제품의 현장 설치·점검·납품을 담당하는 TE팀은 기존에 **엑셀 + 카카오톡 + 수기 문서** 방식으로 업무를 처리했습니다.

| 기존 방식 | 문제점 |
| --- | --- |
| 엑셀 파일 공유 | 버전 충돌, 누락 빈발 |
| 카카오톡으로 인수인계 | 히스토리 추적 불가 |
| 종이 서식 → 수동 PDF | 병원 현장에서 작성 불편 |
| 영업 ↔ TE 소통 단절 | 설치 일정 지연, 정보 누락 |

→ **현장에서 스마트폰 하나로** 모든 문서를 작성하고, Google Sheets에 자동 저장되는 **PWA 시스템**을 직접 기획했습니다.

## ✨ 주요 기능

### 📋 설치환경 정보요청서

-   **영업 ↔ TE 역할 분리 워크플로우**: 영업팀이 A·B·C 섹션 작성 → TE팀이 D·E 섹션 이어서 작성
-   병원명 자동완성 + 주소 자동 불러오기
-   도입 제품명 드롭다운 (Heuron 제품 목록 하드코딩)
-   라이선스 기간 선택 (기준일 + 1~10년 터치 선택 → 만료일 자동 계산)
-   자체 서버 구비 여부 / TE팀 사전 타당성 검토 Y/N 체크
-   작성 완료 시 `.docx` 자동 생성 + Google Sheets 저장

### ✅ 설치 및 운영 적격성 확인서

-   설치 적격성 / 운영 적격성 / 보안환경 항목별 P·F·N 평가
-   **Cloud 타입 선택 시 서버/네트워크 등 6항목 자동 N 변경**
-   제품명·모델명 하드코딩 목록에서 선택 (UDI 제외)
-   현장 사진 첨부 (Google Drive 자동 업로드)
-   서명 패드 → `.docx` 자동 생성

### 🔧 점검 서비스 확인서

-   정기점검 결과 모바일 작성
-   병원 담당자 서명 → `.docx` 자동 생성
-   이메일 자동 발송

### 👥 직원 관리 & 인증

-   `@iheuron.com` 도메인 전용 로그인
-   부서별 역할 분리: 임원 · 영업 · TE · 연구 · 품질
-   역할에 따라 UI 자동 분기

### 📊 대시보드 & 부가 기능

-   CS 이력 조회 / 병원별 제품 현황
-   경비 영수증 사진 업로드
-   알림 센터 (라이선스 만료 등)


## 🚀 배포 방법

### 프론트엔드 (Netlify)

```bash
# 빌드
cd HeuLing
./node_modules/.bin/vite build

# dist 폴더를 Netlify에 드래그앤드롭
# https://app.netlify.com/drop

```

### 백엔드 (Google Apps Script)

1.  [Google Apps Script](https://script.google.com) 새 프로젝트 생성
2.  `Hueing_완성본_v12.gs` 내용 전체 붙여넣기
3.  **배포 \> 웹 앱으로 배포** → 누구나 액세스 허용
4.  생성된 URL을 `src/lib/index.ts`의 `GAS_API_URL`에 입력

## 🔐 환경 변수 설정

`.env` 파일을 루트에 생성하세요:

```env
VITE_GAS_API_URL=https://script.google.com/macros/s/your-deployment-id/exec

```

> ⚠️ `.env` 파일은 `.gitignore`에 포함되어 있습니다. GAS URL을 직접 코드에 노출하지 마세요.

## 🗺 향후 계획 — v13 로드맵

> **설계 확정 (2026-05-18 기준)** · 순차 개발 예정

### v13 핵심 목표

> v12의 TE팀 전용 시스템 → **4개 부서 통합 운영 플랫폼**으로 확장

### 주요 변경사항

| 항목 | v12 (현재) | v13 (예정) |
| --- | --- | --- |
| 사용 부서 | TE팀 단일 | TE · 영업 · 기획 · 관리자 **4부서** |
| 역할 수 | 2종 (TE / 영업) | **5종** (TE / TE\_LEAD / SALES / SALES\_VP / PM) |
| 시트 수 | 8개 | **14개** (신규 6개 추가) |
| GAS 핸들러 | ~30개 | **~48개** (신규 18개 추가) |
| 페이지 수 | ~12개 | **~22개** (신규 10개 추가) |
| 자동 트리거 | 없음 | **4종** (매시간·매일·매주·매월) |
| 워크플로우 | 1종 (CS) | **5종** (데모·CS·계약·클레임·교육) |
| 대시보드 | 없음 | **역할별 5종** |

### 신규 기능 상세

#### 📊 역할별 대시보드 (5종)

-   **TE 대시보드**: 이번 주 CS 건수, 다가오는 설치 일정, 장비 현황
-   **영업 대시보드**: 거래처 현황, 주간 업무 로그, 데모 진행 상태
-   **TE\_LEAD 대시보드**: 팀원 업무 현황, SLA 임박 건, 월간 실적
-   **SALES\_VP 대시보드**: 영업 파이프라인, 계약 현황, 매출 통계
-   **PM(관리자) 대시보드**: 전사 현황 통합 뷰

#### 🔄 5종 워크플로우 엔진

-   **데모 워크플로우**: 데모 신청 → TE 준비 → 진행 → 결과 보고
-   **CS 워크플로우**: 접수 → 배정 → 처리 → 완료 (기존 개선)
-   **계약 워크플로우**: 견적 → 계약서 → 납품 → 검수
-   **클레임 워크플로우**: 접수 → 원인분석 → 처리 → 재발방지
-   **교육 워크플로우**: 신청 → 일정 확정 → 진행 → 수료증

#### 📅 영업팀 기능

-   거래처(병원) 관리 — 담당자, 계약 현황, 방문 이력
-   주간 업무 로그 제출 (매주 금요일 → 자동 취합)
-   코멘트 / 인수인계 메모 기능
-   칸반 보드(드래그앤드롭)로 영업 단계 관리

#### 🤖 자동 보고 시스템

-   **매주 월요일 09:00** — 팀별 주간 현황 자동 이메일 발송
-   **매월 1일 09:00** — 월간 실적 보고서 자동 생성
-   **D-3, D-1** — 라이선스 만료, SLA 임박 리마인더 알림
